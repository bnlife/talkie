# 聊天机器人项目方案

## 一句话

一个桌面聊天机器人，支持多 LLM 厂商、多对话、流式输出、提示词模板、SQLite 存储。

---

## 技术栈

| 层 | 技术 |
|----|------|
| 前端框架 | Vue 3 + TypeScript |
| UI 组件库 | shadcn-vue（Reka UI + Tailwind CSS v4） |
| 状态管理 | Pinia |
| 图标库 | lucide-vue-next |
| 桌面壳 | Tauri v2 |
| 后端 | Rust |
| 数据库 | SQLite（rusqlite） |
| 通信 | Tauri IPC（invoke + event） |
| 测试 | vitest + @vue/test-utils |
| 日志 | tauri-plugin-log（Rust 侧）+ 前端 bridge 转发 |

---

## 架构

### 前端三层

```
bridge/    ← 调 Tauri invoke，纯函数，不持状态
stores/    ← 持有响应式数据（Pinia），调 bridge
pages/     ← 容器组件，串 bridge + store + components
components/← 展示组件，只认 props + events，不知道后端存在
```

### 数据流向

```
组件事件 → dispatch → store → 调 bridge → Tauri invoke → Rust
                                                          → Rust 推 event
                                                          → bridge listener
                                                          → store 更新
                                                          → 组件刷新
```

## 数据模型

### ModelProvider

```typescript
interface ModelProvider {
  id: string
  name: string
  icon?: string
  base_url: string
  api_key: string
  headers: Record<string, string>
  models: string[]
  enabled: boolean
}
```

### Settings

```typescript
interface Settings {
  providers: ModelProvider[]
  active_provider_id: string   // 新建对话时的默认 provider
  temperature: number          // 0 ~ 2，默认 0.7
  top_p: number                // 0 ~ 1，默认 1.0
  last_active_conversation_id?: string
  darkMode?: boolean
}
```

### Conversation

```typescript
interface Conversation {
  id: string
  title: string
  provider_id: string          // 该对话使用的 provider
  model: string                // 该对话使用的模型名
  system_prompt: string
  created_at: number
  updated_at: number
  pinned: boolean
}
```

### Prompt

```typescript
interface Prompt {
  id: string
  name: string
  content: string
  is_default: boolean
  created_at: number
  updated_at: number
}
```

## 页面布局

### 整体结构

```
┌─────────────────────────────────────────────────────────┐
│ Toolstrip │            Header (bg-muted)                │
│ (bg-muted)│  [标题]              [🌙][—][□][×]          │
│           ├─────────────────────────────────────────────┤
│           │ ┌─────────────────────────────────────────┐ │
│           │ │         Content Area (rounded border)    │ │
│           │ │  ┌─────────┬───────────────────────────┐│ │
│           │ │  │ Sidebar │      Main Content         ││ │
│           │ │  │ w-60    │      (bg-background)      ││ │
│           │ │  │(bg-back │                           ││ │
│           │ │  │ ground) │                           ││ │
│           │ │  └─────────┴───────────────────────────┘│ │
│           │ └─────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

### 设计规范

- **外围区域**（Toolstrip + Header）：`bg-muted`
- **内容区域**：`bg-background` + `rounded-lg border` + `p-1` 间距
- **分割线**：不使用布局分割线，仅靠圆角边框分隔
- **hover 效果**：`hover:bg-background`（在 muted 上浮起）或 `hover:bg-foreground/5`（在背景上轻微变化）
- **选中态**：`bg-accent text-accent-foreground`
- **侧栏宽度**：三个页面统一 `w-60`（240px）

### 字号规范

- `text-sm`（14px）：主要内容、标题、输入框、侧边栏、按钮
- `text-xs`（12px）：标签、badge、时间戳、提示文字、区块标题

### 主题色（CSS 变量）

```css
:root {
  --background: hsl(0 0% 100%);      /* 主背景 */
  --foreground: hsl(0 0% 9%);        /* 主文字 */
  --muted: hsl(0 0% 92%);            /* 外围背景 */
  --accent: hsl(0 0% 96%);           /* hover 背景 */
  --border: hsl(0 0% 90%);           /* 边框 */
}
```

## 流式输出

### 发送流

```
前端 invoke("send_message", { conversation_id, content })
  → Rust:
      1. 存 user message 到 SQLite
      2. 查 conversation → 获取 provider_id + model
      3. 从 settings.providers 找对应 provider
      4. 注入 system prompt（对话专属 → 默认模板 → 无）
      5. 构造 HTTP 请求：provider.base_url + api_key + headers + temperature + top_p
      6. 通过 reqwest 发 SSE 请求
      7. 逐行读取响应，每行是一个 chunk
      8. 通过 app.emit("chat:stream-chunk", payload) 推前端
      9. 流结束时 emit("chat:stream-done")
  → 前端监听事件：
      chat:stream-chunk → store 追加 streamingContent
      chat:stream-done  → 归档完整消息
      chat:error        → toast 错误提示
```

### 中断流

```
前端 invoke("stop_stream")
  → Rust 设置 CancellationToken
  → LLM 请求的异步任务检测到取消 → 关闭连接
  → emit stream-done（标记被中断）
```

## SQLite 表结构

### conversations 表

```sql
id            TEXT PRIMARY KEY
title         TEXT DEFAULT '新对话'
provider_id   TEXT DEFAULT ''     -- 绑定的 provider
model         TEXT DEFAULT ''     -- 使用的模型名
system_prompt TEXT DEFAULT ''
created_at    INTEGER NOT NULL
updated_at    INTEGER NOT NULL
pinned        INTEGER DEFAULT 0
```

### messages 表

```sql
id              TEXT PRIMARY KEY
conversation_id TEXT NOT NULL REFERENCES conversations(id) ON DELETE CASCADE
role            TEXT NOT NULL CHECK(role IN ('user', 'assistant', 'system'))
content         TEXT NOT NULL
created_at      INTEGER NOT NULL
token_count     INTEGER
```

### prompts 表

```sql
id          TEXT PRIMARY KEY
name        TEXT NOT NULL
content     TEXT NOT NULL
is_default  INTEGER DEFAULT 0
created_at  INTEGER NOT NULL
updated_at  INTEGER NOT NULL
```
