# 聊天机器人练手项目方案

## 一句话

一个桌面聊天机器人，支持多模型 API、多对话 Tab、流式输出、SQLite 存储。不写 markdown 渲染，不做 RAG。目的是练前端三层架构和 Naive UI 控制。

---

## 技术栈

| 层 | 技术 |
|----|------|
| 前端框架 | Vue 3 + TypeScript |
| UI 组件库 | Naive UI（全量引入，所有交互用 <n-xxx>，不写 CSS） |
| 状态管理 | Pinia |
| 桌面壳 | Tauri v2 |
| 后端 | Rust |
| 数据库 | SQLite（rusqlite） |
| 通信 | Tauri IPC（invoke + event） |
| 日志 | env_logger / tauri-plugin-log，全级别输出 |

---

## 架构

### 前端三层

```
bridge/    ← 调 Tauri invoke，纯函数，不持状态
stores/    ← 持有响应式数据（Pinia），调 bridge
pages/     ← 容器组件，串 bridge + store + components
components/← 展示组件，只认 props + events，不知道后端存在
```

### 数据流向

```
组件事件 → dispatch → store → 调 bridge → Tauri invoke → Rust
                                                          → Rust 推 event
                                                          → bridge listener
                                                          → store 更新
                                                          → 组件刷新
```

## 功能清单

### 第一步：骨架（先跑起来）

Rust 端：搭好项目、定义 models / error / store / config，所有 command 先返回空或 mock

前端：搭好 Vue + Naive UI + Pinia，main.ts 和 App.vue 能渲染，显示一个空窗口

### 第二步：设置页面

SettingsPanel 表单（Naive UI 组件）→ 存到 settingsStore → bridge/settings → Rust 读写 JSON

用户可配：base_url、api_key（敏感字段）、model 下拉选、temperature 滑块

一个"测试连接"按钮，调 Rust → Rust 发 HTTP 请求验证 → 返回成功/失败

### 第三步：聊天

ChatInput 输入文字 → 用户按 Enter 或点发送 → emit send

ChatPage 收到 → 调 bridge/chat.sendMessage() → Rust 收消息 → 调 LLM API（流式）→ 通过 Tauri event 推 chunk

前端 listen event → store 更新 streamingContent → MessageList 逐字追加

流式完成 → 归档到 messages 列表

### 第四步：多对话

Sidebar 显示对话列表 → 新建 / 切换 / 重命名 / 删除

每个对话有自己的 messages 列表，切换 Tab 时 messages 跟着切

---

## 模型管理

内置 DeepSeek 预配置，首次打开直接用。自定义模型：提供一个文本输入区，用户自行写配置（base_url + api_key + model_name），不做各家厂商的适配。Rust 后端统一按 OpenAI-compatible 协议发请求。

切换模型不影响正在进行的流式对话，只影响下一次发送。

## 对话标题

自动生成：收到第一条助手回复后，Rust 后端异步调用 LLM 生成 2~5 字标题，替换默认名。生成失败则保持"新对话"，不阻塞、不报错。

## 流式输出

### 发送流

```
前端 invoke("send_message", { conversation_id, content })
  → Rust:
      1. 存 user message 到 SQLite
      2. 构造 LLM HTTP 请求（含历史消息）
      3. 通过 reqwest 发 SSE 请求
      4. 逐行读取响应，每行是一个 chunk
      5. 通过 app.emit("chat:stream-chunk", payload) 推前端
      6. 流结束时 emit("chat:stream-done")
  → 前端监听事件：
      chat:stream-chunk → store 追加 streamingContent
      chat:stream-done  → 归档完整消息
```

### 中断流

```
前端 invoke("stop_stream")
  → Rust 设置 CancellationToken
  → LLM 请求的异步任务检测到取消 → 关闭连接
  → emit stream-done（标记被中断）
```

### IPC 方案

用 Tauri Event 广播（`app.emit()` + `listen()`），不用 Channel。前端多组件同时监听同一个事件，各自处理。

中断通过 CancellationToken 实现：Rust 后端维护一个中断开关，前端调 `stop_stream` 时触发取消，LLM 请求关闭连接。

切 Tab 不中断流式，后台继续接收，切回来时消息完整。

## SQLite 表结构

### conversations 表

```
id            TEXT PRIMARY KEY
title         TEXT DEFAULT '新对话'
model         TEXT DEFAULT ''
system_prompt TEXT DEFAULT ''
created_at    INTEGER NOT NULL
updated_at    INTEGER NOT NULL
```

### messages 表

```
id              TEXT PRIMARY KEY
conversation_id TEXT NOT NULL REFERENCES conversations(id) ON DELETE CASCADE
role            TEXT NOT NULL CHECK(role IN ('user', 'assistant', 'system'))
content         TEXT NOT NULL
created_at      INTEGER NOT NULL
token_count     INTEGER
```

删除对话时级联删除下属消息。

## Rust 后端设计

### AppState

```rust
struct AppState {
    db: Mutex<Connection>,          // SQLite 连接
    config: Mutex<Settings>,        // 运行时配置
    cancel: Mutex<Option<CancellationToken>>,  // 流式中断
}
```

### 统一错误

```rust
enum AppError {
    DbError(String),         // ERR_db_write, ERR_db_read
    NetworkError(String),    // ERR_net_timeout, ERR_net_connect
    ConfigError(String),     // ERR_config_load, ERR_config_save
    InvalidInput(String),    // ERR_invalid_input
    Internal(String),        // ERR_internal
}
```

所有 command 返回 `Result<T, AppError>`，前端收到错误后显示 toast。

## 日志规范

### 日志文件

```
src-tauri/logs/
├── app.log              ← 全级别（DEBG + INFO + WARN + ERRO）
└── app.error.log        ← 仅 ERRO
    每份 2MB × 10 份滚动
```

### 日志格式

```
[MMDD HH:MM:SS] [级别] 文件名:行号 | 消息 | key=value
```

### 级别定义

| 级别 | 何时打 |
|------|--------|
| TRACE | 噪音级（每帧渲染、chunk 原文）—— 默认关，调试时开 |
| DEBG | 调试用（函数进出、收到 chunk） |
| INFO | 正常操作（用户发消息、配置保存、测试连接） |
| WARN | 预期内异常（重试、降级） |
| ERRO | 非预期错误，必须以 ERR_xxx 开头 |

### 错误码（ERR_全大写 + 全小写英文）

```
ERR_net_timeout     网络超时
ERR_net_connect     连接失败
ERR_db_read         数据库读取失败
ERR_db_write        数据库写入失败
ERR_config_load     配置加载失败
ERR_config_save     配置保存失败
ERR_invalid_input   输入不合法
ERR_internal        未知内部错误
```

## 开发顺序

1. **骨架**：搭项目，Rust 能编译，前端能渲染
2. **设置页面**：配置读写跑通，测试连接跑通
3. **单对话聊天**：发送消息，流式接收，显示纯文本
4. **多对话管理**：侧栏 + Tab 切换，对话 CRUD
5. **打磨**：错误码、日志、测试覆盖
