# 6. MCP 进程生命周期管理：启动后进程立即退出

**文件**: `src-tauri/src/commands/mcp.rs` (toggle_mcp_instance), `src-tauri/src/mcp/runtime.rs` (McpProcess)

**现状**: `toggle_mcp_instance` 在后台线程启动 MCP 进程，spawn + initialize 握手成功（2.6s, 283 bytes），但 McpProcess 立即被 drop/shutdown，`mcp:started` 事件未触发，UI 卡在"启动中"。

**已排除**:
- 并发调用无冲突（测试确认 100 次 spawn+init 无 panic）
- stdout pipe 不是阻塞原因（initialize 响应正常接收）
- 进程启动成功（stderr 输出 `Tavily MCP server running on stdio`）

**未排除**:
- 后台线程本身可能被 drop（需日志确认）
- pool.start() 后立即有其他操作调用 pool.stop()

**触发条件**: 用户在 MCP 页面启动 Tavily 实例时。

**改造方向**: 需在 `McpProcess` 增加运行状态日志，确认后台线程生命周期。

# 7. MCP `read_response` 超时从阻塞循环改为持久化读取线程（已完成）

**文件**: `src-tauri/src/mcp/runtime.rs`

**改造**: `McpProcess` 从 `stdout: Option<BufReader>` 改为 `reader_rx: mpsc::Receiver` + `reader_handle: JoinHandle`。读取线程在进程生命周期内持续运行，`read_response` 通过 `recv_timeout(30s)` 等待。超时或错误时线程 detached（不 join），Drop/shutdown 时先 kill 子进程再 join 读取线程。

**结论**: 完成。解决 per-call spawn 线程导致的 `join()` 死锁问题。
